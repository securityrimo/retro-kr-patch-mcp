# PC-98 국소 사례

여기의 관측은 PC-98 전체의 규칙이 아니다. 발동 조건이 맞을 때 조사 후보로만 사용한다.

## PC98-001

- **관측 범위:** PC-98 마도물어의 `DEMOCTR` 문자열과 게임 고유 Shift_JIS 계열 소비자.
- **사고 맥락:** 표준 CP932로 성공한 pair만 센 통계는 lead `0xEB`를 미사용으로 보고 2-byte direct 한글 슬롯을 564개로 산정했다. 실제 원본에는 표준 디코더가 거부하는 `EB xx` 게임 고유 pair 10종이 세 편에 걸쳐 존재했다.
- **결정 실험:** 파서가 도달 가능한 원본 문자열을 raw pair로 다시 전수 집계해 10쌍을 예약하고, 한글 인코더가 그 pair를 만들지 않는지 검증했다.
- **확정 결론:** 미사용 pair는 표준 디코드 성공 수가 아니라 실제 소비 가능한 raw corpus로 판정해야 했으며, 이 방식으로 게임 고유 pair 10종을 새 공간에서 제외했다.
- **전이 한계:** 정정된 554쌍은 이 리비전의 기존 2-byte direct 공간이며 후에 추가한 4-byte escape의 전체 수용량이 아니다. 부분 lead 재사용도 이 게임의 고유한 해석이며 표준 CP932의 확장 규칙이 아니다.
- **관련 판단 기준:** `references/strategy/font-strategy.md` §2, `references/platforms/pc98.md` §5.

## PC98-002

- **관측 범위:** PC-98 마도물어 A·R·S의 확장 글리프용 JIS→Shift_JIS 변환과 JIS 행 `0x75`·`0x76`의 94-cell 화면 sweep.
- **사고 맥락:** 홀수 JIS 행의 경계식이 cell `0x5F`를 금지된 trail `0x7F`로 만들어 시나리오 글리프 `돌`·`좋`을 손상했다. 별도 컷신 자형 생성기와 Rust 검증기도 같은 off-by-one을 공유했으므로 생성·검증 결과의 일치만으로는 오류를 반증하지 못했다.
- **검증 근거:** 각 cell 위치가 보이는 표식을 한 행의 94개 코드에 넣고 결정적인 전투 메시지 경로로 소비시켰다. 홀수·짝수 행 한 쌍을 각각 94/94 확인하면서 `0x5F→0x7E`, `0x60→0x80` 경계를 바로잡았고, 금지값 전수 검사와 생성 산출물 재검사를 추가했다.
- **확정 결과:** 같은 공식을 공유하는 생성기와 검증기의 합의 대신, 허용 입력 영역의 경계값 검사와 실제 소비자 sweep을 결합해 off-by-one을 해소했다.
- **전이 한계:** 화면에서 94-cell 전수가 확인된 것은 JIS `0x75`·`0x76` 두 행뿐이다. 다른 여덟 행은 같은 산술과 빌드 불변식을 공유하지만 각각 화면 증명됐다고 쓰지 않는다.
- **관련 판단 기준:** `references/strategy/font-strategy.md` §2·§4, `references/strategy/build-and-verify.md` §4·§5.

## PC98-003

- **관측 범위:** PC-98 마도물어 A·R·S의 셰죠 엔딩 크레딧, 64개 문자열을 가리키는 sparse word grid와 1,050바이트 문자열 영역.
- **사고 맥락:** NUL 사이 간격을 개별 고정 슬롯으로 보면 긴 한국어 크레딧 여섯 개를 수용할 수 없었다. 실제 소비자는 문자열을 순서대로 걷지 않고 sentinel이 섞인 grid의 주소에서 각각 진입했다.
- **검증 근거:** 64개 비-sentinel target과 rewrite site를 전수 연결하고 모든 문자열을 경계가 확정된 기존 영역 안에 다시 배치해 grid word를 갱신했다. 810바이트를 사용하면서 overlay 크기와 뒤따른 code를 보존했다. 통제된 런타임 상태에서 재배치된 `프로듀서`를 표시하고 grid index가 다음 셀로 진행함을 확인했다.
- **확정 결과:** 개별 원본 간격 대신 참조 관계가 독립 진입을 보장하는 전체 영역을 재삽입 용량으로 사용해 긴 항목을 축약하지 않고 수용했다.
- **전이 한계:** 포인터 모집단, sentinel 의미, aggregate 경계, cell 폭과 뒤따른 구조가 전수 확정돼야 한다. 통제된 grid 소비 증거는 자연스러운 엔딩→크레딧 전환이나 전체 화면 QA까지 증명하지 않는다.
- **관련 판단 기준:** `references/strategy/reinsertion.md` §1.2·§2·§3, `references/strategy/build-and-verify.md` §3·§4.

## PC98-004

- **관측 범위:** PC-98 마도물어 1편 `MENUDAT1`에서 독립 감사를 통과한 여러 성장 엔트리와 그 뒤의 포인터·디렉터리·구조 주소.
- **사고 맥락:** 성장점 하나씩은 안전해도 앞선 삽입이 만든 출력 좌표를 다음 패치의 입력으로 쓰거나 저장 위치와 타깃에 같은 이동량을 적용하면, 여러 성장을 합친 산출물에서 누적 보정이 빠지거나 중복될 수 있었다.
- **검증 근거:** 모든 패치와 경계 감사를 원본 좌표에 고정하고 성장 패치를 원본 offset 역순으로 splice했다. 포인터 저장 위치와 타깃의 누적 이동을 따로 계산하고 header directory와 구조 tail은 최종 좌표로 한 번만 갱신했다. 실번역 성장 산출물의 구조 재감사와 디스크 재삽입·재추출 일치, 서로 떨어진 두 성장점의 표시·dispatch와 RAM 포인터를 각각 확인했다.
- **확정 결과:** 여러 성장점을 합칠 때 원본 좌표의 불변 작업 계획과 저장·타깃별 누적 변환을 분리하면 위치 의존 구조를 정확히 한 번씩 옮길 수 있었다.
- **전이 한계:** 각 성장 경계의 참조 모집단, 내부 anchor, 구조 주소와 고정 수치의 소유권, 적재 용량이 모두 닫힌 입력에만 적용한다. 제한된 글리프 공급으로 수행한 대표 런타임 검증은 모든 실번역의 자연 표시를 증명하지 않는다.
- **관련 판단 기준:** `references/strategy/reinsertion.md` §1.2·§2·§3, `references/strategy/build-and-verify.md` §3·§4.

## PC98-005

- **관측 범위:** PC-98 마도물어 S의 overlay 문자열과 `mov si, imm16` 참조를 찾는 광범위 byte scan.
- **사고 맥락:** 문자열 경계의 Shift_JIS `82 BE 00 82`에서 trail byte `BE`를 opcode로 읽어 `mov si, 0x8200`이라는 가짜 참조와 중복 suffix 엔트리를 만들었다.
- **검증 근거:** opcode와 immediate 후보가 확정 텍스트 span과 겹치면 제외하고, span 밖의 실제 명령 후보는 확정 문자열 시작을 가리킬 때 유지했다. 합성 경계 반례와 정상 code-site 대조 테스트, 남은 rewrite site가 메시지 span과 겹치면 실패하는 안전 검사를 고정했다.
- **확정 결과:** 문자 인코딩의 일부가 명령 바이트와 같을 수 있는 혼합 영역에서는 바이트 모양만으로 참조를 채택하지 않고 확정된 code·text 소유 경계를 함께 판정해야 했다.
- **전이 한계:** `BE`나 이 Shift_JIS 열을 전역 blacklist로 쓰지 않는다. 실제 문자열 중간 진입 참조가 존재하면 overlap 필터로 버리지 말고 공유 tail·내부 anchor를 명시한 별도 참조 모델로 보존해야 한다.
- **관련 판단 기준:** `references/strategy/text-extraction.md` §1.2·§1.3·§3.5, `references/strategy/reinsertion.md` §2.
